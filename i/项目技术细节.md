# 项目技术细节

## GLES 3.0 着色器转换规则

### 版本声明
- `#version 140` / `#version 330` → `#version 300 es`

### 精度限定符
- 所有 Fragment Shader 添加 `precision highp float;`

### 顶点属性声明
- `in vec3 in_Pos;` → `layout(location = 0) in vec3 in_Pos;`
- `in vec3 in_Nor;` → `layout(location = 1) in vec3 in_Nor;`
- `in vec2 in_UV;`  → `layout(location = 2) in vec2 in_UV;`
- `in vec3 in_Color;` → `layout(location = 3) in vec3 in_Color;`

### 关键修改：mmd.frag 阴影采样
原始桌面 OpenGL 使用 `sampler2DShadow` + `textureProj()`，这在 GLES 3.0 中不被支持。

修改方案：改用 `sampler2D` + 手动深度比较

```glsl
// 原始（桌面 GL）
uniform sampler2DShadow u_ShadowMap0;
visibility = textureProj(u_ShadowMap0, vs_shadowMapCoord[0]);

// 修改后（GLES 3.0）
uniform sampler2D u_ShadowMap0;
vec3 projCoord = shadowCoord.xyz / shadowCoord.w;
float depth = texture(shadowMap, projCoord.xy).r;
float bias = 0.005;
visibility = step(projCoord.z - bias, depth);
```

### 输出变量
- Fragment Shader 使用自定义输出变量（`out vec4 out_Color;` 或 `out vec4 fs_Color;`），而非 `gl_FragColor`
- 原始着色器已全部使用自定义输出变量，无需额外修改

### 保留的特性
- `gl_VertexID` 在 GLES 3.0 中可用（用于 bg.vert 和 quad.vert）
- `texture()` 函数在 GLES 3.0 中与桌面 GL 用法一致
- `discard` 关键字在 GLES 3.0 中可用
- 条件编译 `#ifdef` / `#endif` 在 GLES 3.0 中可用

## GLES_ShaderUtil 设计说明

### 接口
`GLES_ShaderUtil` 与 `GLSLShaderUtil` 具有相同的公开接口：
- `SetShaderDir()`: 设置着色器文件目录
- `SetGLSLDefine()`: 设置 GLSL 预定义宏
- `SetGLSLInclude()`: 设置包含路径
- `CreateProgram(const char* shaderName)`: 加载并编译着色器程序
- `CreateProgram(const char* vsCode, const char* fsCode)`: 从代码字符串编译

### 区别
`CreateProgram(shaderName)` 加载时自动附加 `_gles` 后缀：
- `mmd` → 加载 `mmd_gles.vert` 和 `mmd_gles.frag`
- 而非原始的 `mmd.vert` 和 `mmd.frag`
